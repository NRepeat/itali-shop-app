<objective>
Configure BullMQ queues for entity-specific synchronization, refactor existing Shopify webhook handlers to offload processing to these queues, and create a centralized BullMQ worker to process these jobs.

Purpose: This plan implements the crucial "webhook-to-worker" pattern to prevent Shopify webhook timeouts and enable reliable asynchronous processing, fulfilling the CORE-02 requirement.
Output: `app/service/sync/queues.ts`, `app/service/sync/sync.registry.ts`, modified `app/routes/webhooks.*.tsx` files, and `app/webhook.worker.ts`.
</objective>

    For each identified webhook route file (e.g., `app/routes/webhooks.products.update.tsx`, `app/routes/webhooks.orders.create.tsx` etc.):

       - Extract `topic`, `payload`, and `shop` from `authenticate.webhook(request)`.

       - Immediately add a job to this queue using `await queue.add(topic, { payload, shop });`.
       - Ensure the handler returns `new Response(null, { status: 200 });` *immediately* after adding the job, without any further synchronous processing.
    Avoid: Any synchronous processing of the webhook payload within the HTTP request handler.
  </action>
